Projects

Data Pipeline using Apache Airflow and DBT deployed on AWS ECS. The aim was simulate a real world scenario of data ingestion, transformation, persisting and analysis. Airflow handles the orchestration hence the ingestion was implemented in custom python code with Airflow operators and tasks. With the file sensors in Airflow, this can be extended for streaming for data files. Transformations are handled in Data Build Tool (DBT) and stored in PostgreSQL. By use of a Jenkinsfile, the project is build successfully for deployment and stored on Dockerhub.

Data Streaming using PySpark, Apache Kafka and Tweepy. The streaming projects aims to simulate realtime data streaming and analysis on the fly. The Tweepy package allows us to access tweets based on specific parameters. The tweets are streamed onto a kafka topic, on which by the help of PySpark Structured Streaming, a listener is available for any data arriving. The data is aggregated, transformed and a plot generated for viewing.

LawSMS. A mobile and web application to deliver legal content through SMS subscriptions. Using Django and Django Rest Framework for APIs to be consumed by the mobile and web applications
